# TODO管理アプリ：要素と操作まとめ

## 1. データ要素（クラス）

| クラス名 | 属性 | 説明 |
|----------|------|------|
| Task | title: String<br>dueDate: LocalDate<br>done: boolean<br>priority: int (任意) | タスクそのもの。タイトル、期限、完了状態、優先度を保持 |
| Category (任意) | name: String<br>tasks: List<Task> | タスクを分類するカテゴリ。複数のタスクをまとめる |
| User (任意) | name: String<br>categories: List<Category> | ユーザー情報と所属カテゴリを保持 |

---

## 2. 基本操作（必須）

| 操作 | 説明 |
|------|------|
| タスク追加 | ユーザーが新しいタスクを作成し、リストに追加 |
| タスク表示 | タスクの一覧を表示（完了/未完、タイトル、期限など） |
| タスク完了マーク | タスクを完了状態に変更 |
| タスク削除 | タスクをリストから削除 |

---

## 3. 拡張操作（任意）

| 操作 | 説明 |
|------|------|
| タスクソート | 期限順や優先度順に並べ替え |
| カテゴリ管理 | タスクをカテゴリごとに管理 |
| タスク検索 | タイトルや期限でタスクを検索 |
| データ永続化 | ファイル保存・読み込み（テキスト/JSON） |
| 通知・リマインダー | 期限が近いタスクを通知 |
| GUI表示 | SwingやJavaFXで画面表示 |

---

## 4. ユーザー操作要素

| 操作 | 説明 |
|------|------|
| コマンド入力 | CLIで操作を選択（追加、完了、削除、表示など） |
| タスク選択 | 完了・削除対象のタスクを指定 |
| カテゴリ選択 (任意) | タスクを所属させるカテゴリを選択 |

---

## 5. 内部処理要素

| 処理 | 説明 |
|------|------|
| タスクリスト管理 | ArrayListやLinkedListでタスクを保持 |
| 完了状態更新 | タスクを完了に変更する処理 |
| ソート処理 | 期限順・優先度順に並べ替え |
| データ保存/読み込み | ファイルやJSONへの保存、復元処理 |
