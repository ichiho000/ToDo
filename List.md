# Java ToDo アプリ作成 ToDoリスト

## 1. 要件定義
- [ ] タスクの追加機能を仕様化する  
  - [ ] 全角/半角/数字が入力できることを確認  
  - [ ] 空文字は禁止する  
  - [ ] 最大文字数を決める（任意）
- [ ] タスクの削除機能を仕様化する  
  - [ ] 番号指定で削除  
  - [ ] 存在しない番号の場合のエラー処理を決める
- [ ] 表示機能（一覧）の仕様を決める
- [ ] データ保存は今回は「メモリ上のみ」にするか決定（MVPのため）

## 2. 設計（クラス設計・処理設計）
- [ ] タスクを表す `Task` クラスを作るか検討（文字列だけなら不要）
- [ ] タスクを管理する `TaskManager` クラスを作る  
  - [ ] `addTask(String task)`  
  - [ ] `removeTask(int index)`  
  - [ ] `listTasks()`  
- [ ] メイン実行クラス `Main` を作る  
  - [ ] コンソール入力受付  
  - [ ] メニュー表示（追加 / 削除 / 表示 / 終了）
- [ ] 入力バリデーションの方針を決める  
  - [ ] 全角・半角対応は Java の標準文字列でOK  
  - [ ] 数字・記号も特別な処理不要でOK  
  - [ ] 空文字・スペースのみはエラー

## 3. 仕様に沿った簡易フローチャート作成
- [ ] メニュー選択（1:追加 / 2:削除 / 3:一覧 / 0:終了）
- [ ] 選択に応じて `TaskManager` のメソッドを呼び出し
- [ ] 不正入力はメッセージを出して再入力

## 4. 実装
- [ ] `TaskManager` クラスを実装する
- [ ] メインクラスで Scanner による入力処理を実装する
- [ ] 追加時バリデーション（空文字チェック）
- [ ] 削除時バリデーション（範囲チェック）
- [ ]一覧表示形式を整える

## 5. 動作確認
- [ ] タスクが追加できる  
- [ ] 全角文字（例：買い物）、半角文字（例：study）、数字（123）全て追加できる  
- [ ] タスクが番号指定で削除できる  
- [ ] 存在しない番号でエラー表示  
- [ ] 一覧が正しく表示される  
- [ ] 終了が期待通り動作

## 6. リファクタリング
- [ ] 重複コードの削除  
- [ ] メソッド名・変数名の改善  
- [ ] 例外処理の丁寧さアップ  

## 7. 拡張案（あとから追加）
- [ ] タスクの編集機能  
- [ ] 完了ステータス（完了/未完了）  
- [ ] テキストファイル保存機能
