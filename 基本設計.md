# 🧩 TODO管理アプリ 基本設計書

## 1. システム概要

本アプリは、ユーザーが日々のタスクを登録・確認・完了・削除できるシンプルな **タスク管理ツール（CLI版）** である。  
拡張機能として、カテゴリ管理・ソート・検索・ファイル保存などを追加できる柔軟な設計とする。

## 3. クラス設計

### 3.1 Taskクラス

| 属性 | 型 | 説明 |
|------|----|------|
| title | String | タスクのタイトル |
| dueDate | LocalDate | 期限日 |
| done | boolean | 完了状態（true: 完了 / false: 未完） |
| priority | int | 優先度（1=高, 2=中, 3=低など） |

**主なメソッド**
- `markDone()`: タスクを完了に設定  
- `toString()`: タスク情報を文字列で返す  
- `compareTo(Task other)`: 優先度や期限を比較（ソート用）

---

### 3.2 Categoryクラス（任意）

| 属性 | 型 | 説明 |
|------|----|------|
| name | String | カテゴリ名 |
| tasks | List<Task> | カテゴリに属するタスク一覧 |

**主なメソッド**
- `addTask(Task task)`: タスクを追加  
- `removeTask(String title)`: タスクを削除  
- `getTasks()`: タスク一覧を取得  
- `sortTasksByDueDate()`: 期限順にソート  
- `sortTasksByPriority()`: 優先度順にソート

---

### 3.3 Userクラス（任意）

| 属性 | 型 | 説明 |
|------|----|------|
| name | String | ユーザー名 |
| categories | List<Category> | 所属するカテゴリ |

**主なメソッド**
- `addCategory(Category category)`: カテゴリ追加  
- `removeCategory(String name)`: カテゴリ削除  
- `getAllTasks()`: 全カテゴリのタスク一覧を取得  

---

## 4. 基本操作フロー

| 操作 | 入出力 | 説明 |
|------|---------|------|
| タスク追加 | 入力：タイトル、期限、優先度<br>出力：登録完了メッセージ | 新規タスクを作成しリストに追加 |
| タスク表示 | 入力：なし<br>出力：一覧表示 | すべてのタスクを表示（完了状態・期限・優先度） |
| タスク完了マーク | 入力：タスク番号またはタイトル<br>出力：更新メッセージ | 該当タスクの `done` を `true` に変更 |
| タスク削除 | 入力：タスク番号またはタイトル<br>出力：削除メッセージ | 指定されたタスクを削除 |

## 8. CLI操作イメージ
==== TODO管理アプリ ====
1. タスク追加
2. タスク一覧表示
3. タスク完了マーク
4. タスク削除
5. 終了
選択 > 1
タイトルを入力: レポート提出
期限を入力 (yyyy-mm-dd): 2025-11-10
優先度 (1=高 2=中 3=低): 1
→ タスクを追加しました！
